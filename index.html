rejit_parse

Parse a string. 

rejit_parse_result rejit_parse(
    const char *str,
    rejit_parse_error *err, 
    rejit_flags flags);
Parameters
str

The string to parse.

err

The location to write errors to.

flags

The flags to use during parsing.

Discussion

Parse the given string, returning a rejit_parse_result. If an error occurs, store the information in the memory pointed to by err. Otherwise, err->kind will be set to RJ_PE_NONE.

int rejit_search(
    rejit_matcher m,
    const char *str,
    const char **tgt,

tgt

typedef struct rejit_group_type { 
    const char *begin, *end; 
} rejit_group;
Fields
begin

A pointer to the beginning of the match.

end

A pointer to the end of the match.

Discussion

This is the type that is used to store groups that were matched in the input. Note that both pointers are to the original string. Therefore, the original must not be deleted for these pointers to remain valid.

rejit_flags

Compile flags. 

typedef enum { 
    RJ_FNONE = 1<<0, 
    RJ_FICASE = 1<<1, 
    RJ_FDOTALL = 1<<2, 
    RJ_FUNICODE = 1<<3, 
} rejit_flags;
Constants
RJ_FNONE

No flags.

RJ_FICASE

Case insensitive matching.

RJ_FDOTALL

Make dot (.) also match newlines.

RJ_FUNICODE

Make character classes Unicode-aware.

Discussion

The different flags you can pass to the *compile* functions. Enum values can be bitwise OR'd with other ones to compile various flags.

rejit_matcher

A compiled regex. 

typedef struct rejit_matcher_type { 
    rejit_func func; 
    size_t sz; 
    int groups; 
    rejit_flags flags; 
}* rejit_matcher;
Fields
groups

The number of groups that the compiled regex requires.

flags

The flags the regex was compiled with.

Discussion

This is the type that is returned when compiling a regular expression. All the fields should be treated as an internal implementation detail except for groups.

rejit_parse_error_kind

The kind of error that occurred.

typedef enum { 
    RJ_PE_NONE, 
    RJ_PE_SYNTAX, 
    RJ_PE_UBOUND, 
    RJ_PE_OVFLOW, 
    RJ_PE_RANGE, 
    RJ_PE_INT, 
    RJ_PE_LBVAR, 
    RJ_PE_MEM, 
} rejit_parse_error_kind;
Constants
RJ_PE_NONE

Successful parse/no error.

RJ_PE_UBOUND

Unbound parenthesis, bracket, or curly brace.

RJ_PE_OVFLOW

Stack overflow (too many nested parens).

RJ_PE_RANGE

Bad character range.

RJ_PE_INT

Expected an integer.

RJ_PE_LBVAR

Lookbehind contains variable-length expression.

RJ_PE_MEM

Out of memory.
